# SPEC: 機關分析

> 反向工程文件（2026-04-24 補）。位置：`app.py` tab4「機關」，L308-413。

## 1. Objective
- 看機關類型分佈（中央/地方/學校/醫療/國營/其他）
- 下鑽單一機關：看這機關歷史案件、服務過的廠商、需求主題、凌網表現
- Top 50 機關排行

**User**：業務，研究目標客戶機關；主管，分析市場結構。

## 2. Commands
`streamlit run app.py` → tab4「機關」。

## 3. 內容
1. **機關類型分布**：2 個 pie chart（案件數佔比、預算佔比）
2. **機關下鑽**：
   - selectbox「機關類型篩選」→ 縮小候選池
   - selectbox「選機關」
   - 選定後顯示：
     - 4 metrics：總案數/決標案數/決標總額/平均單案
     - 案件類型分布 + Top 5 標的分類
     - **服務此機關的所有廠商**（含擅長領域 Top 3、自家標示）
     - 需求主題分布（水平 bar）
     - 凌網在此機關承接提示（若有）
     - 案件清單（排序日期降序）
3. **機關 Top 50**：案件數 / 決標數 / 總預算

## 4. 廠商擅長領域算法
從**整體資料集**（非僅此機關）對每家廠商：
- explode companies → 取該廠商所有歷史 titles
- 套 `count_themes()` 依 `DEMAND_THEMES` 關鍵字命中
- 取 Top 3 主題顯示如「系統維運(12), AI/數據(5), 網站平台(3)」

## 5. Boundaries
- 機關類型分類規則在 `helpers.py:UNIT_TYPE_RULES`
- 廠商擅長領域用整體資料，不限此機關（避免樣本太小）
